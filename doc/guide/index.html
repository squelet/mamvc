Quick guide to µ-AMVC

Trio - The MVC pattern

VIEW: DOM Builder

TRIO is a web MVC technology, so the presentation (VIEW) layer is at the end represented as HTML Dom. Therefore it needs to allow fully flexible way of creating and manipulating the DOM tree of the page.

µ-AMVC is pure Ecma Script 5 based library, so the API is JavaScript. For DOM manipulation it uses a builder pattern per element. Each builder instance wraps a DOMNode, and provides methods to simply and intuitively manipulate it.

Quick dynamic page

In TRIO MVC library, you create dynamic and interactive pages very simply with clean separation of responsibilities.

  • View is created using the view builder
  • Any dynamic aspect is delegated to model (state), which can be directly used when building view
  • Model (state) updates are triggered by commands or by model dependencies.

Let's demonstrate it on very quick example:

1. Prepare models import {state} from "../../trio.js" let clickCounter = state(0) 2. Add view which will display the state, and dynamically update it import {body, div} from "../../trio.js"; body().add(div().add(clickCounter))
3. Add element, which updates the counter model import {body, a} from "../../trio.js"; body().add(a().onClick(() => clickCounter.set(clickCounter.get() + 1)).add('Click here'))
